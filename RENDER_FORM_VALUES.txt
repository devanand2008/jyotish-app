Important:
If Render gives a different service URL, replace jyotish-astro-app.onrender.com with your real Render URL everywhere above.

If Google still shows redirect_uri_mismatch:
Open error details on that Google page, copy the exact redirect_uri shown there, and add that exact full URL to Authorized redirect URIs.
For phone login, test with the Render HTTPS URL, not a local LAN IP address.

Do not add private IP URLs such as:
http://10.60.238.176:8080/login.html
http://192.168.1.5:8080/login.html

Google Cloud rejects these because OAuth redirect URIs need a public domain such as onrender.com, ngrok-free.app, trycloudflare.com, .com, .org, etc.
